# 进程间通信 IPC（Inter-Process Communication）

> 进程彼此隔离，但有时又需要合作，所以需要支持进程间通信。

**方法总览：**

+ 低级通信：只能传递状态和整数值（控制信息），包括进程互斥和同步所采用的信号量和管程机制。缺点：
    + 信息量小
    + 编程复杂
    + 适用范围受限：低级通信机制往往要求通信双方处于共享地址空间环境
+ 高级通信：主要有三类。
    + 管道
    + 共享内存
    + 消息系统

## 管道（Pipe）

分为无名管道和有名管道两种。

### 无名管道 (Pipe)

> 管道是操作系统提供的一个内核缓冲区，用于让一个进程把数据传给另一个进程。

**特点：**

1. **半双工：**数据通常单向流动（A 写，B 读）若双向通信，通常建立两个管道。
2. **亲缘进程使用：**通常用于父子进程、兄弟进程（通过 fork 继承文件描述符）。
3. **先进先出（FIFO）：**先写入的数据先被读出。
4. **存在于内存中：**管道不是普通磁盘文件，而是内核中的临时通信通道。

以下是我们可能会在 bash 中用到的管道操作

```bash
awk '{print $1}' "$access_src" | sort | uniq -c | sort -nr | awk '{print $2, $1}' > $summary_dst
```

下面是一个更能体现管道是两个进程间的通信的例子：

```bash
while true; do
    echo hello
    sleep 1
done | head -n 5
```

**说明：**

Shell 会同时启动两个进程：

左侧进程：持续输出 hello
右侧进程：head -n 5 读取前 5 行

左侧进程的输出通过管道传给右侧进程作为输入。
当右侧读满 5 行后退出，管道关闭，左侧进程也结束。

### 有名管道（Named Pipe，又称FIFO）

无名管道通常用于具有亲缘关系的进程（如父子进程、兄弟进程）。

为了解决这一限制，操作系统提供了**有名管道**：它在文件系统中拥有一个路径名，不相关的进程只要能够访问该路径，就可以通过它进行通信。

**特点：**

1. **有名字：** 以文件路径形式存在，可通过 `mkfifo` 创建。  
2. **无亲缘关系也可通信：** 不要求进程之间存在父子关系。  
3. **先进先出（FIFO）：** 先写入的数据先被读出。  
4. **本质仍是管道：** 数据通常在内核缓冲区中流动，而不是长期保存到磁盘。
