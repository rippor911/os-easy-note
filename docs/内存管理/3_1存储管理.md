# 存储管理

## 一、背景：

+ 一般而言，存储空间越大，访问所需时间越长
+ 内存性能的提升速度落后于CPU
+ **最佳状态**应是各层次的存储器都处于均衡的繁忙状态

## 二、基本机制：抽象 

**地址空间：**一个进程所能够用于访问内存的地址集合

**逻辑地址**（虚拟地址）：程序所使用的地址

**物理地址：**物理内存中数据存储的地址

**原则：** 独立（程序只需管虚拟地址） + 保护（其他程序无法访问本程序的地址空间）

## 三、功能

+ **存储分配和回收**：存储管理的主要内容
+ **地址变换**：可执行文件生成中的链接技术、程序加载时的重定位技术，进程运行时硬件和软件的地址变换技术和机构
+ **存储共享和保护**：代码和数据共享，对地址空间的访问权限（读、写、执行）
+ **存储器扩充**：涉及存储器的逻辑组织和物理组织

 由应用程序控制：覆盖；

由OS控制：交换（整个进程空间），请求调入和预调入（部分进程空间）

## 四、方法

引入碎片概念为评价优劣提供参考：

+ **碎片：** 内存中无法被利用的存储空间
+ **内碎片：** 指分配给作业的存储空间中未被利用的部分
+ **外碎片：** 系统中无法利用的小的空闲分区，是造成内存系统性能下降的主因

+ 单道程序：静态地址，速度快，但是大程序无法加载，浪费资源
+ 多道程序：分区式分配，固定（静态）分区 or 可变（动态）分区

|          | 优点             | 缺点                                         | 方法                                         |
| -------- | ---------------- | -------------------------------------------- | -------------------------------------------- |
| 固定分区 | 易于实现，开销小 | **内碎片**造成浪费，分区总数固定，限制并发数 | 单一队列，多队列                             |
| 动态分区 | 没有内碎片       | 有外碎片                                     | 位图表示法（分区表），链表表示法（分区链表） |

**分区分配操作：** 取最小分区（大小为size），若空闲分区分给请求分区之后剩余大小小于size，则一并分给请求的程序（减少外碎片）

**分区回收操作：** 若有相邻空闲分区，则合并；否则，新建表项放入空闲分区表

下面是**基于顺序搜索的分配算法**：

|                       | 定义                                   | 说明                 |
| --------------------- | -------------------------------------- | -------------------- |
| 首次适应（First Fit） | 每次从头找第一个满足的空闲区           | 碎片多，查找也慢     |
| 下次适应（Next Fit）  | 从上次查找结束的地方找下一个满足空闲区 | 空闲均匀，但比较碎   |
| 最佳适应（Best Fit）  | 总是找最小的满足空闲区（即≥且最接近）  | 碎片非常小，难以利用 |
| 最坏适应（Worst Fit） | 总是找最大的空闲区                     | 后来的大作业难满足   |

下面是**基于索引搜索的分配算法**：

|                          | 定义                                                         | 说明                                                     |
| ------------------------ | ------------------------------------------------------------ | -------------------------------------------------------- |
| 快速适应（分类搜索）     | 常用大小的空闲区设立单独的空闲区链表，为多个空闲链表设立管理索引表 | 查找效率高，无碎片，但算法复杂，系统开销大，存在浪费     |
| 伙伴系统（buddy system） | 在分配存储块时将一个大的存储块分裂成两个，这两个即为伙伴（同个父亲） | 利用计算机二进制数寻址优点加速，属于固定分区（有内碎片） |

**紧凑技术（Compaction）：**

+ 通过移动作业，把多个分散的小分区拼接成一个大分区的方法

+ 目标：消除外碎片
+ 时机：大作业请求时，找不到大空闲分区，但空闲分区总容量量足够
+ 方法：动态重定位

**多重分区分配：** 为了支持结构化程序设计，操作系统往往把一道作业分成若干片段，这样，片段之间不需要连续

**分区的存储保护：**

+ 界限寄存器方法：如上下界、基址、限长
+ 存储保护键方法：存储块（锁）+ 作业（钥匙）

## 五、覆盖交换技术

**问题：** 程序运行过程可能会增长需要的空间

**解决方案：** **覆盖**与**交换**技术

> 覆盖与交换可以解决在小的内存空间运行大作业的问题，是“扩充”内存容量和提高内存利用率的有效措施。

**覆盖（Overlay）：**

+ 定义：把一个程序划分为一系列功能相对独立的程序段，让执行时不要求同时装入内存的程序段组成一组覆盖段，共享主存同一区域的内存扩充技术
+ 说明：早期 OS 使用，从时间维度扩充空间
+ 缺点：对用户不透明，增加用户负担（用户必须考虑如何划分覆盖段）

**交换（Swapping）：**

+ 定义：把暂时不用的程序或数据从主存移到辅存，再读入指定程序，将控制转交给它
+ 优点：增加并发程序数；给用户提供适当响应时间；不影响程序结构（用户无需操心）
+ 缺点：增加处理器开销（控制换入换出）；对程序整个地址空间传送，忽略地址访问的统计特性（例如局部性原理等）
+ 选择原则：换出等待 I/O 的进程
+ 交换时机：有程序很少再用，内存空间不足时
+ 工作：保存现场，创建新线程的运行现场

|          | 覆盖                 | 交换                             |
| -------- | -------------------- | -------------------------------- |
| 作用范围 | 减少一个程序运行空间 | 可让整个程序暂存于外存，让出空间 |
| 实现方式 | 由程序员指定覆盖结构 | 不需要程序员给出覆盖结构         |
| 作用对象 | 同一作业或程序       | 作业之间或程序之间               |
