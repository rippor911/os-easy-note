# 段式管理

## 动机

> [!NOTE]
> 页式管理把内存切成固定大小的页，方便系统管理，但不符合程序的逻辑结构。  
> 程序天然由代码、数据、栈、堆等部分组成，因此引入分段。

1. **方便编程**

程序通常由多个逻辑部分组成：

- 主程序
- 子程序 / 函数库
- 数据区
- 栈区

分段后，每段对应一个独立逻辑单位，可按“段号 + 段内偏移”访问。

2. **更容易共享**

共享通常是共享一个完整模块，例如：

- 文本编辑器代码
- 数学库
- 排序库

分页下一个模块可能分散在多个页中，共享管理复杂。  
分段后可直接共享整个代码段。

3. **更容易保护**

不同逻辑部分权限不同：

- 代码段：只读 + 可执行
- 数据段：可读写
- 栈段：可读写，不可执行

按段设置权限比按页更自然。

4. **支持动态增长**

某些区域运行时会增长：

- 栈：函数调用加深时增长
- 堆：malloc / new 时增长
- 编译器符号表、中间结构增长

分段更符合“整块区域增长”的需求。

---

## 基本思想

一个作业的地址空间由若干段组成，每段是一组相对完整的逻辑信息。

例如：

- 0段：代码段
- 1段：数据段
- 2段：栈段

每段从0开始编址，长度可不同。

逻辑地址表示为：

```text
(段号, 段内位移)
````

例如：

```text
(1, 200)
```

表示第1段内偏移200字节处。

因此：

* 页式管理：一维地址空间
* 段式管理：二维地址空间

## 地址变换

> [!NOTE]
> 段表记录“段号 -> 段基址、段长”的映射关系。
> 段表存放在内存中，基址和长度由段表寄存器给出。

访问一次数据通常需要两次访存：

1. 查段表
2. 访问实际内存

### 地址变换过程

设逻辑地址为：

```text
(S, d)
```

其中：

* `S`：段号
* `d`：段内位移

步骤：

1. 检查段号 `S` 是否越界
   若 `S >= 段表长度`，则越界中断。

2. 读取第 `S` 个段表项，得到：

* 段基址 `base`
* 段长 `limit`

3. 检查段内位移是否越界

```text
d >= limit
```

若成立，则越界中断。

4. 形成物理地址

```text
物理地址 = base + d
```

## 信息共享

例：

一个系统同时接纳40个用户，都运行文本编辑程序：

* 代码区：160KB
* 数据区：40KB

若不共享：

```text
40 × (160 + 40) = 8000KB
```

若共享代码段：

```text
160 + 40 × 40 = 1760KB
```

节省大量内存。

> [!NOTE]
> 共享代码要求代码是 **可重入代码（Reentrant Code）**

### 可重入代码要求

1. 不使用全局 / 静态可变变量
2. 不修改代码自身
3. 不调用其他不可重入代码

---

## 优缺点分析

### 优点

1. 易于实现共享
2. 易于按逻辑单位保护
3. 更好支持动态增长
4. 更符合程序员思维方式

### 缺点

1. 地址变换需查段表，速度变慢
2. 段长不固定，产生外碎片
3. 可能需要内存紧凑
4. 大段需要连续空间，分配困难

---

### 与页式管理的区别

| 项目     | 页式管理    | 段式管理       |
| ------ | ------- | ---------- |
| 单位     | 页（固定大小） | 段（长度可变）    |
| 性质     | 物理单位    | 逻辑单位       |
| 地址空间   | 一维      | 二维         |
| 用户是否可见 | 否       | 是          |
| 优势     | 消除外碎片   | 共享、保护、逻辑清晰 |

## 段页式管理

### 基本思想

段页式管理 = 分段 + 分页。

基本思想是：

- 用分段方法来分配和管理虚拟地址空间
- 用分页方法来分配和管理物理内存

也就是说，先把用户程序按逻辑划分成若干段，再把每个段划分成若干页。

例如：

- 代码段 → 分成若干页
- 数据段 → 分成若干页
- 栈段 → 分成若干页

这样既保留了分段的逻辑结构，又利用分页避免连续分配带来的外碎片问题。

---

### 地址结构

段页式管理中的逻辑地址由三部分组成：

```text
(段号 S, 段内页号 P, 页内位移 W)
````

含义：

* `S`：访问哪个段
* `P`：访问该段中的哪一页
* `W`：访问该页中的哪个字节

因此，段页式地址空间仍然体现分段的逻辑结构，但段内部采用分页管理。

---

### 表结构

段页式系统中同时设置段表和页表。

#### 段表

每个进程有一张段表。

段表项通常包含：

* 段号
* 该段页表始址
* 该段页表长度

段表的作用是找到某个段对应的页表。

#### 页表

每个段有一张页表。

页表项通常包含：

* 页号
* 状态位
* 物理块号

页表的作用是找到某个段内页面对应的物理页框。

---

### 地址变换过程

设逻辑地址为：

```text
(S, P, W)
```

地址变换过程如下：

1. 从 PCB 中取出段表始址和段表长度，装入段表寄存器。

2. 将段号 `S` 与段表长度比较。

```text
若 S >= 段表长度，则产生越界中断
```

3. 若段号合法，则根据段表始址和段号 `S` 找到对应段表项。

4. 从段表项中取出该段的：

* 页表始址
* 页表长度

5. 将段内页号 `P` 与页表长度比较。

```text
若 P >= 页表长度，则产生越界中断
```

6. 若页号合法，则根据页表始址和页号 `P` 找到对应页表项。

7. 从页表项中取出物理块号。

8. 将物理块号与页内位移 `W` 拼接，得到物理地址。

```text
物理地址 = 物理块号 + 页内位移
```

---

### 访存次数

段表和页表都存放在内存中。

因此，在没有快表的情况下，访问一次数据或指令通常需要访问内存三次：

1. 访问段表
2. 访问页表
3. 访问实际数据或指令

为了提高速度，通常需要设置高速缓存或快表。

---

### 与二级页表的区别

段页式和二级页表都需要多次查表，但目的不同。

二级页表：

```text
页目录号 -> 页表号 -> 页内偏移
```

它本质上还是分页，只是为了减少页表占用空间。

段页式：

```text
段号 -> 段内页号 -> 页内偏移
```

它先按程序逻辑分段，再在段内分页。

所以：

* 二级页表解决“大页表如何组织”的问题
* 段页式解决“程序逻辑分段 + 物理内存分页分配”的问题

---

### x86 中的段页式地址映射

Intel x86 的地址映射可以看作两阶段：

```text
逻辑地址 -> 线性地址 -> 物理地址
```

第一阶段：段式地址映射

```text
逻辑地址 = 段选择子 + 段内偏移
```

CPU 根据段选择子查找段描述符，得到段基址、段界限和访问权限。

然后：

```text
线性地址 = 段基址 + 段内偏移
```

第二阶段：页式地址映射

CPU 再根据页目录和页表，把线性地址转换成物理地址。

```text
线性地址 -> 页目录 -> 页表 -> 物理页框 -> 物理地址
```

因此，x86 的地址转换过程可以理解为：

```text
分段机制负责：逻辑地址 -> 线性地址
分页机制负责：线性地址 -> 物理地址
```

---

### 小结

段页式管理的核心是：

```text
按段组织程序，按页分配内存
```

它结合了两者优点：

* 分段：方便共享、保护、动态增长，符合程序逻辑
* 分页：离散分配物理内存，减少外碎片

但代价是地址变换更复杂，需要同时查段表和页表。
