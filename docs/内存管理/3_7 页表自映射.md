# 页表自映射

> 我们知道，是操作系统OS来填写页表项，以反映内存布局，控制内存，而页表放在物理内存中，而OS通过虚拟地址访存

那么，OS 想修改页表项时，如何“找到它的虚拟地址”？？

case1:单独设置一块虚拟地址空间放页表

**case2:把页表放在物理地址对应的虚拟地址空间中** → 这就是页表自映射采用的思想

如果把页表放在虚拟地址空间中，一定会存在一个**“不动点”**，可以这么理解：

+ 拿着北京地图的北京游客，一定能在地图上找到一点和自己当前所处的位置对应

> [!IMPORTANT]
>
> 以二级页表自映射为例（取页面大小 $2^kByte$，页表项大小 $2^nByte$ ,虚拟地址空间为 $m$ 位）：
>
> $PDE_{self-mapping}:$ 自映射页目录项虚拟地址
>
> $PD_{base}:$ 页目录基地址（虚拟空间）
>
> $PT_{base}:$ 页表基地址（虚拟空间）
>
> $PT_{base} = (PT_{base} >> (2n-k))<<(2n-k)$ 	        // 所有页表在同一个页目录页中（假设为页 $page$ ）
>
> $PD_{base}=PT_{base}|((PT_{base}>>n)<<k)$			  // 页目录是 $page$ 对应页表的一个页
>
> $PDE_{self-mapping}=PD_{base}|(PD_{base}>>n<<k)$
>
> 还是搞不明白。。。

**优点：**

+ 便于OS找到页表项对应的虚拟地址，以便于修改页表项内容
+ 对于多级页表，相比于临时映射节省了访问物理内存次数开销

**缺点：**安全性问题（如果硬编码，被知道了很危险，让$PT_{base}$动态变化解决这个问题）

> [!CAUTION]
>
> 采用页目录自映射后，所映射的物理内存量变小，因为页目录占用了页表项，被占用的页表项不能使用
