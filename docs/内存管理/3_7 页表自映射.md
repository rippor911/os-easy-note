# 页表自映射

> 我们知道，是操作系统 OS 来填写页表项，以反映内存布局、控制内存；而页表本身放在物理内存中，OS 平时又是通过虚拟地址访存。

那么，OS 想修改某个页表项时，如何“找到这个页表项的虚拟地址”？

一种直观做法是：单独保留一块内核虚拟地址空间，把需要修改的页表页临时映射进去。

页表自映射采用的是另一种更固定的做法：**让页目录中的某个 PDE 指向页目录自己所在的物理页框**。这样一来，页目录和所有已经存在的页表页，都会出现在一段确定的虚拟地址窗口里。

如果把“描述映射关系的数据结构”也映射进它自己描述的虚拟地址空间，就会出现一个类似 **“不动点”** 的位置。可以这么理解：

+ 拿着北京地图的北京游客，一定能在地图上找到一点和自己当前所处的位置对应。

> [!IMPORTANT]
>
> 初学的时候，如果搞不清楚什么行为是 `MMU` 做的，什么行为是 `OS` 做的，就很容易把虚拟空间和物理空间混淆。
>
> 这里的目标不是改变普通地址的访存流程，而是让 OS 能够方便地用虚拟地址访问并修改页目录项、页表项。
>
> 在自映射机制里主要关心几个地址：
>
> - $PT_{base}$：页表窗口的起始虚拟地址，页表窗口指的是所有的页表在虚拟空间中连续排列所占的区域
> - $PD_{base}$：页目录所在页的虚拟地址
> - $PDE_{self}$：的页目录项所在的虚拟地址

## 32 位两级页表中的地址转换

以 32 位地址空间、4 KiB 页、每个页表项 4 Byte 为例：

- 虚拟地址高 10 位 `va[31:22]` 是页目录索引
- 中间 10 位 `va[21:12]` 是页表索引
- 低 12 位 `va[11:0]` 是页内偏移
- 一个页目录或页表页正好有 1024 个表项

用虚拟地址 `va` 访问内存时，**`MMU`** 做：

1. 从页目录基址寄存器（如 x86 的 `CR3`）获取页目录基地址，这个地址是物理地址。
2. 根据 `va[31:22]` 找到对应页目录项（PDE），读取该 PDE，得到页表所在的物理页框号。
3. 根据页表物理页框号找到页表页，再根据 `va[21:12]` 找到对应页表项（PTE），读取目标物理页框号。
4. 用目标物理页框号加上 `va[11:0]` 作为偏移（offset），得到最终物理地址并完成访问。

而 **`OS`** 构建自映射时，做的是：

1. 选一个页目录项作为自映射项。哪一个都可以，比如说我们选最后一个，即 `pgdir[1023]`。
2. 把这个 PDE 的物理页框号填成页目录自己所在的物理页框号。
3. 刷新 TLB，或者让相关 TLB 项失效。

这一步完成后，自映射窗口居然就建立起来！它只是让“页目录”和“页表页”有了一组稳定的虚拟地址。尚未分配的页表页，对应窗口里仍然没有有效映射（也就是说没有填入物理页框号和权限位）。

## 为什么能找到页表项

> 以小见大

假设 $PDE_{self-mapping}$ 是第 $r$ 个页目录项：

- 页表窗口起始地址：$PT_{base} = r << 22$
- 页目录所在页地址：$PD_{base} = (r << 22) | (r << 12)$
- 自映射 PDE 的地址：$PDE_{self-mapping} = PD_{base} + r \times 4 = (r << 22) | (r << 12) | (r << 2)$

> [!IMPORTANT] 分析
> 先记住一句话：自映射页目录项在页目录中的相对位置，相当于页目录页在整个页表窗口的相对位置，相当于页表窗口在整个虚拟内存空间的相对位置
>
> 前提：页表窗口是 4MB 对齐，是 OS 可决定，具体原因在理解了流程后其义自见
>
> 重点看如何找页目录：
> 页表窗口 4MB 对齐 → `[31:22]` 相同 → 页表窗口内的任意虚拟地址（当然也包括页目录） 都是第 r 个 4MB 中 → MMU 找到第 r 个页目录项（$PDE_{self-mapping}$），读到页目录物理页框
> 对于其他普通页表页，下一步正常由 MMU 读取即可
> 对于页目录项 → 页目录页是整个页表窗口的第 r 个页表页 → 第 r 个“页表”的第 r 个“页表项”正是 $PDE_{self-mapping}$ ! → 读取到的物理页框是页目录页框 → 根据 offset 可访问到相应页目录项

对于任意虚拟地址 `va`，它对应的 PTE 虚拟地址可以写成：

```text
PTE_addr = PT_base + va[31:22] * 4096 + va[21:12] * 4
```

```text
PTE_addr = PT_base | (va[31:22] << 12) | (va[21:12] << 2)
           ↑                ↑                   ↑
    页表窗口基址  1个页表管 4MB ，这里     一个页表项管 4KB，
                 算出是第几个页表      这里算出是这个页表的第几个页表项
```

也就是说，OS 把一个 PDE 覆盖的 4 MiB 虚拟地址区间拿出来，专门作为“页表窗口”。在这个窗口中，连续摆放着最多 1024 个页表页。

## 优点和缺点

**优点：**

+ OS 可以用简单公式计算任意 PTE 的虚拟地址，修改页表项更方便。
+ 不需要每次修改页表时都额外建立临时映射，内核代码更简单。
+ 对多级页表也可以推广：通过递归映射，可以为各级页表建立可访问的虚拟窗口。

**缺点：**

+ 会占用一个 PDE 覆盖的虚拟地址区间。32 位、4 KiB 页、两级页表下，通常是 4 MiB 虚拟地址空间不能再作为普通映射使用。

> [!CAUTION]
>
> 采用页目录自映射后，虽然物理内存容量没有减少，但是相当于用一个页表页作为页目录页，也就是说，少了一个可映射用户/普通虚拟地址空间的页表页。
