# 页表自映射

> 我们知道，是操作系统OS来填写页表项，以反映内存布局，控制内存，而页表放在物理内存中，而OS通过虚拟地址访存

那么，OS 想修改页表项时，如何“找到它的虚拟地址”？

case 1:单独设置一块虚拟地址空间放页表

**case 2:把页表放在物理地址对应的虚拟地址空间中** → 这就是页表自映射采用的思想

如果把页表放在虚拟地址空间中，一定会存在一个**“不动点”**，可以这么理解：

+ 拿着北京地图的北京游客，一定能在地图上找到一点和自己当前所处的位置对应

> [!IMPORTANT]
>
> 注意，初学的时候，如果搞不清楚什么行为是 `MMU` 做的，什么行为是 `OS` 做的（或者说视角不同），就很容易把虚拟空间和物理空间混淆，接下来的分析里会着重标明这一点。

以 32 位地址空间为例

用虚拟地址 `va` 访问内存时，**`MMU`** 做：

1. 从页目录基地址寄存器获取页目录基地址（物理地址）
2. 根据 `va[31:22]` 找到第 `va[31:22]` 个页目录项，读取这个页目录项得到页表对应的物理页框号
3. 根据 2 中的物理页框号找到对应的物理页框，也就是我们要找的页表基地址，找到第 `va[21:12]` 个页表项，读取物理页框号
4. 根据 3 中的物理页框号得到物理页的基地址，然后 `va[11:0]` 作为偏移，成功访问到内存

而 **`OS`** 在尝试构建自映射机制时，是已知页目录基地址 `base` 对应的虚拟地址 `pgdir`，下面从这个起点出发：

**优点：**

+ 便于OS找到页表项对应的虚拟地址，以便于修改页表项内容
+ 对于多级页表，相比于临时映射节省了访问物理内存次数开销

**缺点：**安全性问题（如果硬编码，被知道了很危险，让$PT_{base}$动态变化解决这个问题）







> [!CAUTION]
>
> 采用页目录自映射后，所映射的物理内存量变小，因为页目录占用了页表项，被占用的页表项不能使用
