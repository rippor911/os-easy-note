# 请求式分页系统

> [!NOTE]
>
> + **进程的逻辑空间：**
> + 一个进程的逻辑空间的建立是通过链接器（Linker），将构成进程所需要的所有程序及运行所需环境，按照某种规则装配链接而形成的一种规范格式(布局)
> + **交换分区（交换文件）：**
> + 一段连续的磁盘空间（按页划分的），并且对用户不可见。它的功能就是在物理内存不够的情况下，操作系统先把内存中暂时不用的数据，存到硬盘的交换空间，腾出物理内存来让别的程序运行（Linux中为swap，Windows中则以文件pagefile.sys形式存在）

**核心思想：** 只需把**当前需要的部分页面**调入主存

+ 调入什么：OS核心部分、正在运行的用户进程相关程序和数据
+ 何时调入：OS在系统启动时调入；用户程序取决于调入策略（预调页，按需调页，$\dots$）
+ 如何调入：缺页错误处理机制
